32-bit vs 64-bit

What is the difference between 32-bits and 64-bits. First of all, what is a bit? To put it very simply, a bit is a binary instruction that can be understood by a computer. In our case a bit is either 0 or 1, off or on, yes or no.

To keep things as simple as possible let's look at the binary representation of the decimal numbers up to 8.

0001 = 20 = 1

0010 = 21 = 2

0100 = 22 = 4

1000 = 23 = 8

Starting from right to left, the '1' digit represents the presence of a power of 2 corresponding with the postion of the digit in binary code.

0001 = 0(23)+0(22)+0(21)+1(20)

0010 = 0(23)+0(22)+1(21)+0(20)

0100 = 0(23)+1(22)+0(21)+0(20)

1000 = 1(23)+0(22)+0(21)+0(s0)

As you see above we can create composite integers from 0 - 15 by combining the various bit elements with 0000 = 0 and 1111 = 15. In most computer circles you may see 4-bits called a nibble or even nybble. Mostly you won't see this term, as most in the industry prefer the 8-bits to the byte as the prefered basic grouping of bits.

Now what all this talk about 32-bit versus 64-bit, what is the true difference? To put it simply, it is about "address" size. So on a 32-bit machine, the size computer componants can process and handle data that is up to 32-bits in length. So when your computer is presenting an image file it is processing the data in 32-bit long chunks at a time. 32-bit numbers can represent integers from 0 all the way up to 4,294,967,295. WoW! That is a rather large range of numbers, yet sadly we soon realized, we were running out of numbers

On the other end we have 64-bit architecture, which is the modern bus architecture for most new computers being sold today. The same paradign and pattern applies to 64-bit registers. This allows data flows as wide as 64 bits wide. This gives us a range from 0 all the way up to... 18,446,744,073,709,551,615. That is over 18 thousand trillion numbers!! Double WoW WoW!!
